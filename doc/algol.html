Algol 68 Genie - An Algol 68 subset interpreter

Algol 68 Genie (Algol68G) is a fairly fully featured Algol 68 implementation.

Algol68G's front-end parses an entire Algol 68 source program. If parsing is successful, the syntax tree (that serves as an intermediate program representation) is interpreted by Algol68G's back-end.

  • Many runtime checks (for example uninitialised values, overflow, accessing NIL, bounds checks on rows and dynamic scope checks).
  • Optional graphical procedures using the GNU Plotting Utilities, for drawing in X Windows, postscript format, pseudo graphical interface format (pseudo-gif) and portable any-map format (pnm) from Algol 68.
  • Optional numerical procedures using the GNU Scientific Library.
  • LONG and multiprecision LONG LONG arithmetic modes.
  • Format texts, straightening and formatted transput.
  • UNIX extensions that allow for instance for executing child processes that communicate through pipes.

Consult the GNU General Public License for details.

  • Download Algol68G Mark 4, a gzipped tar archive of ±370 kB containing C source code and documentation. Algol68G Mark 4 was developed on GNU/Linux (ppc) and was regression-tested on FreeBSD (ix86).
  • Download a Win32 executable of Algol68G Mark 4 for Microsoft Windows, which is made available by Theo Vosse. This gzipped tar archive of ±700 kB contains the executable and documentation. Unzip (using f.i. WinZip) the archive to obtain a68g.exe which can be executed from command prompts command.com or cmd.exe. The executable is linked to the GNU Plotting Utilities.
  • For Debian GNU/Linux, Sian Leitch has posted a binary package.
  • The included manual can also be found on-line.
  • This manual includes an Algol68G syntax summary.
  • Download the regression test set for Algol68G Mark 4. This gzipped tar archive of ±675 kB contains more than 700 test programs, including The Revised MC Algol 68 Test Set.

New features

Algol68G Mark 4 (June 2004)

  • More GSL support.
  • Modes LONG BITS and LONG LONG BITS.
  • By default, accept (..) as alternative for [..].
    Under control of option brackets, (..), [..] and {..} are equivalent.
  • Accept loop clause as enclosed clause.
  • Accept united modes whose components are related through deflexing.
  • Thread-safe plotutils interface.
  • Dynamic scope checking.
  • Simplified core management.
Algol68G Mark 3.2 (March 2004)
  • Overflow checks for primitive modes.
Algol68G Mark 3.1 (January 2004)
  • Optional system stack overflow handling.
  • Improved multiprecision library.
Algol68G Mark 3 (September 2003)
  • Formatted transput.
  • Improved speed of multiprecision library.
Algol68G Mark 2 (March 2003)
  • Mode BYTES and LONG BYTES.
  • Mode FORMAT and parsing of FORMAT texts.
  • Straightening in unformatted transput.
  • UNIX system level IO.
  • Mode PIPE and UNIX support.
Algol68G Mark 1 (November 2002)
