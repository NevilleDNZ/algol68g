History

When IFIP Working Group 2.1 discussed a successor for Algol 60 in 1965, Van Wijngaarden presented his proposal Orthogonal design and description of a formal language (available from Karl Kleine's collection). This paper formed the basis from which WG 2.1 worked on Algol X, which would develop into Algol 68. Wirth left the project to work on Algol W and later on Pascal.

Algol 68 was defined in a formal document, first published in January 1969, and later published in Acta Informatica and also printed in Sigplan Notices. Algol 68 was the first major language for which a full formal definition was made before it was implemented.

Probably because of the formal defining document, which requires some study to comprehend, the misconception got hold that Algol 68 is a complex language. In fact, Algol 68 is a rather lean orthogonal language, that even today is a beautiful means for denoting algorithms.


Readily available Algol 68 documentation

  • This manual contains an Algol68G syntax summary for quick reference.
  • Wilhelm Kloke has made available an on-line version of the formal Revised Report.
  • Sian Leitch wrote a good informal textbook on Algol 68: Programming Algol 68 Made Easy.
  • Karl Kleine collects and posts Algol 68 historic documents.
  • Brian Wichmann maintains The Algol Bulletin on-line archive.
  • Dick Grune has posted an Algol 68 program collection.

Algol 68 implementations

Current other implementations

  • Algol 68 to C translator (a68toc), a ported compiler by Sian Leitch. The front-end essentially is the Algol 68 RS portable compiler originally written by the Defence Research Agency when it was known as the RSRE (Royal Signals and Radar Establishment). Only that part of Algol 68 needed for the ELLA compiler was implemented by a68toc with the result that formats are unavailable. Algol 68 RS did not implement the parallel-clause.
  • Algol 68S compiler by Charles Lindsey.
  • Other options from Algol 68 web resources (maintained by Greg Nunan), or the Algol 68 directory from the Open Directory Project.

Former implementations

See for instance [AB51.3.1] or [Koster 1994]:
  • ALGOL68R. Currie et al., RSRE (Royal Signals and Radar Establishment, Malvern) 1970. One-pass subset, used extensively in military and scientific circles in the UK on ICL machines.
  • CONTROL DATA ALGOL 68. Zoethout et al., CDC Netherlands 1974. Full implementation, used mainly in teaching in Germany and the Netherlands on CDC mainframes.
  • ALGOL68S. Hibbard et al., UK and Carnegie Mellon 1976. Subset, ported to small machines as the PDP-11, (apparantly) mostly used in teaching.
  • ALGOL68RS. For ICL 2900, VAX/VMS and other systems.
  • FLACC Full Language Algol 68 Checkout Compiler. Mailloux et al., Edmonton, Canada 1978. An interpreter for the full language, available on IBM mainframes and used in teaching, both in North America and in Europe.
  • ALGOL68C. Bourne et al., Cambridge 1980. Subset implementation for IBM mainframes, DECsystems 10 and 20, VAX/UNIX, CYBER 205 and other systems.
